Como a economia é calculada (sem overclaim).
  • Tokens lidos = soma do pico de crescimento de cada sessão (o quanto de conversa você arrastaria se nunca desse /clear+handover).
  • Economia (tokens) = por sessão, pico − custo de retomar (nunca negativo). O baseline (system+tools+memória) é pago dos dois jeitos, então não entra como economia.
  • R$ = tokens × (preço input USD/1M do modelo) × câmbio. Preços da tabela oficial da API (Opus US$5, Sonnet US$3, Haiku US$1, Fable US$10 por 1M input); o modelo é auto-detectado do log e o câmbio é seu.
  • ⚠️ Contexto re-arrastado é re-lido como input — por isso o preço de input. Mas re-leitura costuma cair no cache (~0,1× do input), então o R$ aqui é um teto (pior caso, sem cache).
  • Números de uma máquina, do seu próprio log — não é promessa, é medição.
